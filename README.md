# vime — Vim Explorer

`vime` is a fast, keyboard-driven terminal file manager written in Go. It brings vim-style navigation to your file system, letting you browse directories, manage files, and launch your editor without ever leaving the terminal.

## Features

- vim-inspired keybindings (`h`/`j`/`k`/`l` navigation)
- Fuzzy search and normal substring search
- File operations: copy, rename, delete, create files and directories
- Copy file path to clipboard
- Tab completion in command mode
- Open files in your preferred editor
- Lightweight TUI built with [Bubble Tea](https://github.com/charmbracelet/bubbletea)

## Requirements

- [Go](https://go.dev/dl/) **1.21 or later** (the module was developed with Go 1.26.1)
- A Unix-like terminal (Linux, macOS, WSL)

## Installation & Build

### Clone the repository

```bash
git clone https://github.com/duckisam/vime.git
cd vime
```

### Download dependencies

```bash
go mod download
```

### Build the binary

```bash
go build -o vime .
```

This produces a `vime` executable in the current directory.

### (Optional) Install system-wide

```bash
go install github.com/duckisam/vime@latest
```

Or copy the built binary to a directory on your `PATH`:

```bash
cp vime /usr/local/bin/vime
```

## Usage

Launch `vime` from any directory:

```bash
vime
```

`vime` opens an interactive file browser starting in your current working directory. When you quit, it prints the last visited directory path to stdout, which can be used for shell integration (see [Shell Integration](#shell-integration)).

## Keybindings

### Navigation

| Key | Action |
|-----|--------|
| `j` | Move cursor down |
| `k` | Move cursor up |
| `l` or `Enter` | Enter directory / confirm action |
| `h` | Go to parent directory |
| `c` | Change to an arbitrary directory (command prompt) |

### File Operations

| Key | Action |
|-----|--------|
| `Y` | Copy selected file path to clipboard |
| `r` | Rename selected file or directory |
| `d` | Create a new directory |
| `%` | Create a new file |
| `D` | Delete selected file or directory |

### Editor & Modes

| Key | Action |
|-----|--------|
| `e` | Open selected file in the configured editor |
| `:` | Enter command mode |
| `/` | Enter normal (substring) search mode |
| `f` | Enter fuzzy search mode |

### General

| Key | Action |
|-----|--------|
| `q` or `Ctrl+C` | Quit |

## Configuration

Configuration lives in `internal/config/`.

### Change the default editor

Edit `internal/config/commands.go` and update `EditorCommand`:

```go
var (
    EditorCommand = "nvim"  // change to "vim", "nano", "hx", etc.
)
```

Then rebuild:

```bash
go build -o vime .
```

### Keybindings

All keybinding constants are defined in `internal/config/keys.go`. Modify the constants there and rebuild to remap any key.

### Styles

Colour and styling configuration is in `internal/config/styles.go`, using the [Lip Gloss](https://github.com/charmbracelet/lipgloss) library.

## Shell Integration

Because `vime` prints the last visited directory to stdout on exit, you can integrate it with your shell so that your terminal session changes into the directory you were browsing.

### Bash / Zsh

Add the following function to your `~/.bashrc` or `~/.zshrc`:

```bash
vcd() {
    local dest
    dest=$(vime)
    if [ -n "$dest" ] && [ -d "$dest" ]; then
        cd "$dest"
    fi
}
```

Then call `vcd` instead of `vime` when you want the `cd` effect.
